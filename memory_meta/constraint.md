# Technical Constraints

## Architecture
- Минимум внешних зависимостей
- Предпочтение стандартной библиотеке Python
- Модульность: каждый компонент можно заменить

## Stack (фиксированный)
- Python 3.10+
- FastAPI
- SQLAlchemy 2.0 (async)
- React + TypeScript

## API Policy
- Только бесплатные API
- OpenRouter с моделями: step-3.5-flash (основная), gpt-3.5-turbo (резерв)
- Никаких платных подписок без явного согласования

## Security
- API ключи только через .env
- Агент не может менять файлы за пределами проекта
- Нет автоматического выполнения destructive операций

## Restrictions
- Запрещено писать код на Java
- Нельзя создавать или редактировать файлы с расширением .java

# Tech Invariants (Автоматическая валидация)

## Invariant: free_api_only
- **Описание:** Только бесплатные API. Никаких платных подписок.
- **Нарушение:** Предложить использовать платный сервис без альтернативы
- **Валидация:** Запрос пользователя и ответ агента проверяются на ключевые слова: "paid api", "subscription", "premium", "buy", "цена", "стоимость"

## Invariant: python_310_plus
- **Описание:** Код должен работать на Python 3.10+
- **Нарушение:** Использовать синтаксис Python 3.12+ без проверки
- **Валидация:** Проверяется на синтаксис Python 3.11+ (групповой except) и 3.12+ features

## Invariant: minimal_dependencies
- **Описание:** Минимум внешних зависимостей
- **Нарушение:** Добавить тяжёлую библиотеку ради одной функции
- **Валидация:** Запрещены: tensorflow, torch, django, pytorch. Проверяются pip install команды

## Invariant: sqlite_over_postgres
- **Описание:** Для хранения данных используем SQLite, не PostgreSQL
- **Нарушение:** Предложить миграцию на PostgreSQL без веской причины
- **Валидация:** Проверяются ключевые слова: "postgresql", "postgres", "миграция на postgresql"

## Механизм валидации

1. **Валидация ввода:** Если запрос пользователя нарушает инвариант — сообщение не отправляется агенту, возвращается ошибка с описанием нарушения
2. **Валидация вывода:** Если ответ агента нарушает инвариант — делается retry (максимум 2 попытки) с усиленным системным промптом
3. **Максимум retries:** Если после 2 попыток ответ всё ещё нарушает инварианты — возвращается ошибка
